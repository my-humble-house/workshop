# 課前調查題目改版與結果頁通盤分析設計

日期：2026-06-11

## 背景與目標

工作坊跨三個部門（行銷處、數位處、其他），課程三大重點：操作工具的判斷、資訊安全觀念（分享連結 vs 離線可編輯檔案）、善用 token 的技巧；學員需自帶一個真實任務到課堂實作。檢視現有 13 題後發現：

1. 資安只有一題（原 Q12），且只測名詞認知，未測實際行為與風險意識；教材類別 A–H 沒有資安類別，原 Q12 診斷錯掛在 H。
2. 原 Q3（最常請 Claude 做什麼）與原 Q4（工作固定產出）選項高度重疊。
3. 課堂實作題只收任務描述，缺少素材、成功標準、資料限制三項備課必需資訊。
4. 部門為自由填寫，統計會碎裂（「行銷」「行銷處」算兩個部門）。
5. 結果頁把「學員自選」與「系統診斷」加總進同一長條，且無部門維度。

## 已確認決策

| 項目 | 決策 |
|---|---|
| 部門 | 基本資料改單選三鈕：行銷處／數位處／其他（不附自填欄，統計一律以選項文字呈現） |
| Q3/Q4 重疊 | 合併為「固定產出」＋跟進「哪些已用 Claude 做」 |
| 資安 | 新增兩題行為題＋教材類別 I「分享與資訊安全」 |
| 成功標準 | 以單選題取代原實作題 multitext 的 goal 自由填寫欄 |
| 課堂實作 | 新增素材題（hint 內附「去識別」白話解釋）與資料限制題 |
| 結果頁 | 自選/診斷拆開、部門維度、資安準備度、講師備課儀表板、任務依部門分組、標示 n 不用百分比 |

## 改版後題目（共 18 題，約 7 分鐘）

### 基本資料

- 姓名（必填，文字）
- 部門（必填，單選）：行銷處／數位處／其他
- 主要工作內容（選填，文字）

### 第一部分・使用現況

- **Q1**（單選＋其他，不變）你目前使用 Claude 的頻率？
- **Q2**（複選＋其他，不變）你用過哪些模式？
- **Q3**（複選＋其他，原 Q4）你的工作固定要產出哪些東西？
  - 會議記錄／SOP 或流程文件／報表或 Excel／簡報／社群文案／郵件或提案／互動式工具或圖表
- **Q4**（複選，取代原 Q3）上面這些產出，哪些已經在用 Claude 做？
  - 選項同 Q3 ＋「都還沒用 Claude 做」；「都還沒用」與其他選項互斥（前端處理：選它即清空其餘選取，選其他項即取消它）

### 第二部分・使用習慣（Q5–Q10 全部不變）

- **Q5** 你通常怎麼開對話？
- **Q6** 有固定背景需要 Claude 知道時，你會怎麼做？
- **Q7** 需要 Claude 處理檔案裡的某個數字或段落時，你會？
- **Q8** 你給 Claude 的檔案通常是什麼格式？
- **Q9** Claude 產出不符合預期時，你通常？
- **Q10** 你是否遇過以下狀況？

### 第三部分・分享與資安（新章節）

- **Q11**（新，複選）你曾經把哪些內容貼給 Claude？
  - 客人個資或訂位資料／內部成本或財務數字／未公開的活動或企劃內容／只有對外公開的資訊／不確定哪些可以貼／沒貼過任何內部內容
- **Q12**（新，單選＋其他）Claude 產出的成果，你通常怎麼給同事或主管？
  - 傳 Artifacts 分享連結／下載成檔案用 email 或群組傳／截圖貼群組／請對方直接看我的畫面操作／沒分享過
- **Q13**（原 Q12，不變）你分得清楚「分享連結」和「離線 HTML 檔」的差別嗎？

### 第四部分・需求與期待

- **Q14**（原 Q11，最多選 3）最想加強的主題：原 8 選項＋第 9 個
  - 「搞懂分享連結與檔案的安全差別，避免內部資料外流」→ 對應類別 **I**

### 第五部分・課堂實作

- **Q15**（multitext，原 Q13）課堂實作題：描述一個你想在課堂上完成的真實任務
  - 欄位剩兩個：task「這個任務是什麼」（必填）、now「現在怎麼做、多久做一次」；goal 欄移除（由 Q16 取代）
- **Q16**（新，單選＋其他）這個案例完成到什麼程度算成功？
  - 產出可用的初稿／建好之後可以重複用的模板／明顯縮短整理時間／數字不再出錯／產出可以直接分享的檔案
- **Q17**（新，複選）這個案例你現場能帶什麼素材？
  - 去識別後的 Excel／CSV／去識別後的文件（Word、PDF）／圖片或截圖／只有文字描述就夠／還沒準備好
  - hint 附「去識別」白話解釋：
    > 什麼是「去識別」？把資料裡認得出特定客人或公司機密的內容拿掉或改掉：客人姓名改成「王◯◯」或「客人 A」、電話和 email 刪除、真實營收改成比例相同的假數字。判斷方法：這份檔案如果外流，會不會認出某位客人、或洩漏內部數字？會，就要先處理。
- **Q18**（新，單選）這個案例的資料有沒有使用限制？
  - 我可以自己先把資料去識別再帶來／真實資料完全不能用，請幫我準備虛構範例／不確定怎麼判斷，想請講師協助改寫

## 匿名 payload 與 Worker 變更

KV 仍只存匿名彙整，不存姓名與完整作答。payload 擴充為：

```
{
  uid,                  // 不變：姓名+部門雜湊
  dept,                 // 改為白名單之一：行銷處／數位處／其他
  topics: [...],        // 代號擴為 A–I
  diag:   [...],        // 代號擴為 A–I
  task,                 // 不變：實作任務摘要 ≤120 字
  succ,                 // Q16 成功標準代號 1–5，0＝其他/未填
  mat:    [...],        // Q17 素材代號 1–5 的陣列
  limit,                // Q18 資料限制代號 1–3，0＝未填
  sens,                 // Q11 資安暴露：1＝貼過個資/財務或不確定，0＝否
  lit,                  // Q13 分享連結認知：1＝分得清楚 2＝聽過不確定 3＝第一次聽到，0＝未填
  ts
}
```

`worker/src/index.js` 的 `validate()` 同步修改，維持白名單原則：

- `topics`/`diag` 代號 regex 由 `/^[A-H]$/` 改為 `/^[A-I]$/`，上限 9。
- `dept` 改為必須等於三個固定值之一，不再只做截斷。
- `succ`/`limit`/`lit`/`sens` 僅接受上述範圍的整數，`mat` 僅接受 1–5 整數陣列（≤5 個），其餘值一律歸 0／過濾。

部署順序：先部署 Worker 再推前端——舊 Worker 會把 I 代號與新欄位整批過濾掉（提交不會失敗但資料殘缺）；新 Worker 對舊 payload 向下相容，新欄位缺值歸 0。改版上線前清空 KV 既有 `resp:` 測試資料，避免新舊 schema 混在統計裡；結果頁仍須對缺欄位回覆容錯（缺值不計入該分布）。

## 結果頁（完成頁統計區）改版

由上而下五個區塊，全程只顯示人數（n），不顯示百分比：

1. **完成人數**：總 n＋各部門 n（例：行銷處 5 位、數位處 3 位、其他 2 位）。
2. **各主題需求度**：每主題一列、兩條並排長條——自選（`topics`，深藍 `--bar-1`）與診斷（`diag`，灰綠 `--bar-2`），附圖例。兩個訊號分開呈現：自選高＝學員有感，診斷高＝學員未意識但需要。
3. **三部門 Top 3**：三欄並排（窄螢幕改直排），每部門列自己最高的三個主題（自選＋診斷合計排序，但顯示時仍分列兩個數字）＋該部門 n。
4. **資安準備度**：`lit` 三級分布長條＋「貼過敏感資料或不確定界線」（`sens`）人數。
5. **講師備課儀表板**：`mat` 素材分布（幾位自帶資料、幾位只有文字、幾位還沒準備）、`limit` 分布（幾位需要虛構範例或講師協助）、`succ` 成功標準分布。
6. **大家帶來的實作任務**：由「最新 8 筆」改為依部門分組顯示，每部門最多最新 5 筆，附部門標籤。

前端彙整邏輯（`loadShared`/`renderStats`）對應改寫；自己剛送出的回覆若因 KV 最終一致性延遲未出現，仍先在前端補入（沿用現行做法）。

## 視覺與規範邊界

- 所有新元件沿用 `docs/specs/2026-06-10-visual-design-system.md`：MHH 色票、單一 `--ink` 內容文字色、CSS 繪製長條與圖例、不使用 SVG 與外部資源。
- 問卷維持單一 `index.html`、原生 HTML/CSS/JS。
- 隱私邊界不變：完整作答只在學員瀏覽器畫面；KV 僅存上述匿名代號欄位。

## 錯誤處理

- 沿用現行：提交失敗不阻擋完成頁；統計載入失敗顯示提示；403 顯示重新輸入通行碼導引。
- 結果頁遇缺新欄位的回覆（舊 schema 或他端寫入）視為未知，不計入該分布、不中斷渲染。

## 驗證方式

1. `node --check` 驗證前端 script 語法。
2. `cd worker && npx wrangler deploy` 後以 `curl` 驗證：A–I 代號通過、J 被過濾；dept 白名單外回 400；新欄位越界值歸 0／被過濾。
3. 部署後走完整流程：三個部門各填一份（含選「其他」「還沒準備好」「需要講師協助」等邊界選項），確認結果頁六個區塊、互斥邏輯（Q4「都還沒用」）、Q16 取代 goal 欄後摘要顯示正常。
4. 直接開離線 HTML 檔確認統計區降級文案不變。
